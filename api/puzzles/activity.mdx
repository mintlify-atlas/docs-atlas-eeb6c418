## Endpoint

```bash
GET https://lichess.org/api/puzzle/activity
```

## Description

Download your puzzle activity in [ndjson](#ndjson) format. Each line represents a puzzle round, including the puzzle details and whether it was solved correctly.

Puzzle activity is streamed as [ndjson](#ndjson) (newline-delimited JSON).

## Authentication

<Warning>
This endpoint requires authentication with OAuth2 or API token.
</Warning>

Required OAuth2 scopes:
- `puzzle:read`
- `web:mod` (for mobile clients)

## Query Parameters

<ParamField query="max" type="integer" default="unlimited">
  Maximum number of puzzle rounds to return. Must be at least 1.
  
  If not specified, all puzzle activity will be streamed.
</ParamField>

<ParamField query="before" type="integer">
  Return only puzzle activity before this timestamp (in milliseconds since epoch).
  
  Can be used for pagination by passing the timestamp of the oldest entry from a previous request.
</ParamField>

<ParamField query="since" type="integer">
  Return only puzzle activity since this timestamp (in milliseconds since epoch).
  
  Useful for fetching only recent activity.
</ParamField>

## Response Format

The response is streamed as [ndjson](#ndjson) (newline-delimited JSON). Each line is a JSON object representing one puzzle round.

<ResponseField name="date" type="integer" required>
  Unix timestamp in milliseconds when the puzzle was attempted
</ResponseField>

<ResponseField name="win" type="boolean" required>
  Whether the puzzle was solved correctly on the first attempt
</ResponseField>

<ResponseField name="puzzle" type="object" required>
  The puzzle information
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique puzzle identifier
    </ResponseField>
    
    <ResponseField name="fen" type="string">
      The FEN string of the puzzle position after the initial move
    </ResponseField>
    
    <ResponseField name="rating" type="integer">
      The puzzle difficulty rating
    </ResponseField>
    
    <ResponseField name="plays" type="integer">
      Total number of times this puzzle has been attempted
    </ResponseField>
    
    <ResponseField name="solution" type="array">
      Array of UCI move strings representing the solution
    </ResponseField>
    
    <ResponseField name="themes" type="array">
      Array of theme keys describing the puzzle's tactical patterns
    </ResponseField>
    
    <ResponseField name="lastMove" type="string">
      The last move played in UCI format before the puzzle position
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl https://lichess.org/api/puzzle/activity?max=10 \
  -H "Authorization: Bearer {your_api_token}"
```

```bash cURL with date range
curl "https://lichess.org/api/puzzle/activity?since=1640000000000&max=50" \
  -H "Authorization: Bearer {your_api_token}"
```

```javascript JavaScript
const response = await fetch(
  'https://lichess.org/api/puzzle/activity?max=10',
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_TOKEN'
    }
  }
);

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const lines = decoder.decode(value).split('\n');
  for (const line of lines) {
    if (line.trim()) {
      const round = JSON.parse(line);
      console.log(round);
    }
  }
}
```

```python Python
import requests
import json

headers = {'Authorization': 'Bearer YOUR_API_TOKEN'}
response = requests.get(
    'https://lichess.org/api/puzzle/activity',
    params={'max': 10},
    headers=headers,
    stream=True
)

for line in response.iter_lines():
    if line:
        puzzle_round = json.loads(line)
        print(puzzle_round)
```
</CodeGroup>

## Rate Limiting

This endpoint is subject to standard Lichess API rate limits:
- Authenticated requests: More generous limits
- The stream is throttled to ~20 puzzle rounds per second
- Total concurrent connections are limited per IP and user

## NDJSON Format

The response uses newline-delimited JSON (ndjson) format:
- Each line is a complete JSON object
- Lines are separated by newline characters (`\n`)
- Process line-by-line for memory efficiency
- Stream can be consumed in real-time
