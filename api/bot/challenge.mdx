## Overview

Bots can create challenges to other players, and receive and respond to challenges from other players. Challenges are received through the [event stream](/api/bot/playing#stream-events).

---

## Accept a Challenge

### POST /api/challenge/{challengeId}/accept

Accept an incoming challenge.

#### Authentication

Requires OAuth token with `challenge:write`, `bot:play`, or `board:play` scope.

#### Parameters

<ParamField path="challengeId" type="string" required>
  The challenge ID received from the event stream
</ParamField>

<ParamField query="color" type="string">
  Accept the challenge and request a specific color: `white` or `black`. If not specified, color is random.
</ParamField>

#### Request

<CodeGroup>
```bash cURL
curl -X POST https://lichess.org/api/challenge/kZ4Zs7cN/accept \
  -H "Authorization: Bearer <your_token>"

# Request white
curl -X POST "https://lichess.org/api/challenge/kZ4Zs7cN/accept?color=white" \
  -H "Authorization: Bearer <your_token>"
```

```python Python
import requests

def accept_challenge(challenge_id, token, color=None):
    headers = {"Authorization": f"Bearer {token}"}
    url = f"https://lichess.org/api/challenge/{challenge_id}/accept"
    
    params = {}
    if color:
        params['color'] = color
    
    response = requests.post(url, headers=headers, params=params)
    
    if response.status_code == 200:
        print(f"Challenge {challenge_id} accepted")
        return True
    else:
        error = response.json()
        print(f"Error: {error.get('error', 'Unknown error')}")
        return False

# Accept with random color
accept_challenge("kZ4Zs7cN", "lip_yourtoken")

# Accept and request white
accept_challenge("kZ4Zs7cN", "lip_yourtoken", color="white")
```

```javascript JavaScript
const token = "lip_yourtoken";

async function acceptChallenge(challengeId, color = null) {
  const url = new URL(`https://lichess.org/api/challenge/${challengeId}/accept`);
  if (color) {
    url.searchParams.append('color', color);
  }
  
  const response = await fetch(url, {
    method: "POST",
    headers: { "Authorization": `Bearer ${token}` }
  });
  
  if (response.ok) {
    console.log("Challenge accepted");
    return await response.json();
  } else {
    const error = await response.json();
    console.error("Error:", error);
    throw error;
  }
}

await acceptChallenge("kZ4Zs7cN");
```
</CodeGroup>

#### Response

<ResponseField name="ok" type="boolean">
  Success status
</ResponseField>

<CodeGroup>
```json Success (200)

}
```


#### Rate Limits

Bots can accept challenges but are subject to:
- Maximum concurrent games (typically 5 for bots)
- Opponent-specific limits to prevent spam

## Decline a Challenge

### POST /api/challenge/{challengeId}/decline

Decline an incoming challenge.

#### Authentication

Requires OAuth token with `challenge:write`, `bot:play`, or `board:play` scope.

#### Parameters

<ParamField path="challengeId" type="string" required>
  The challenge ID to decline
</ParamField>

<ParamField body="reason" type="string">
  Reason for declining. Options:
  - `generic` - No specific reason
  - `later` - Will play later
  - `tooFast` - Time control too fast
  - `tooSlow` - Time control too slow
  - `timeControl` - Don't like the time control
  - `rated` - Don't want to play rated
  - `casual` - Don't want to play casual
  - `standard` - Only play standard chess
  - `variant` - Don't play this variant
  - `noBot` - Don't play with bots
  - `onlyBot` - Only play with bots
</ParamField>

#### Request

<CodeGroup>
```bash cURL
curl -X POST https://lichess.org/api/challenge/kZ4Zs7cN/decline \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{"reason":"later"}'
```

```python Python
import requests

def decline_challenge(challenge_id, token, reason="generic"):
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    }
    url = f"https://lichess.org/api/challenge/{challenge_id}/decline"
    data = {"reason": reason}
    
    response = requests.post(url, headers=headers, json=data)
    
    if response.status_code == 200:
        print(f"Challenge {challenge_id} declined")
        return True
    else:
        print(f"Error declining challenge")
        return False

# Decline with reason
decline_challenge("kZ4Zs7cN", "lip_yourtoken", reason="tooFast")
```
</CodeGroup>

#### Response

<ResponseField name="ok" type="boolean">
  Success status
</ResponseField>

<CodeGroup>
```json Success (200)
{
  "ok": true
}
```
</CodeGroup>

## Create a Challenge

### POST /api/challenge/{username}

Challenge another player to a game.

#### Authentication

Requires OAuth token with `challenge:write`, `bot:play`, or `board:play` scope.

#### Parameters

<ParamField path="username" type="string" required>
  Username of the player to challenge
</ParamField>

<ParamField body="rated" type="boolean" default="false">
  Whether the game should be rated
</ParamField>

<ParamField body="clock.limit" type="integer">
  Clock initial time in seconds (for real-time games)
</ParamField>

<ParamField body="clock.increment" type="integer">
  Clock increment in seconds (for real-time games)
</ParamField>

<ParamField body="days" type="integer">
  Days per turn (for correspondence games, 1-15)
</ParamField>

<ParamField body="color" type="string" default="random">
  Your color: `white`, `black`, or `random`
</ParamField>

<ParamField body="variant" type="string" default="standard">
  Chess variant:
  - `standard`
  - `chess960`
  - `crazyhouse`
  - `antichess`
  - `atomic`
  - `horde`
  - `kingOfTheHill`
  - `racingKings`
  - `threeCheck`
</ParamField>

<ParamField body="fen" type="string">
  Custom initial position in FEN notation
</ParamField>

<ParamField body="keepAliveStream" type="boolean" default="false">
  Keep the stream open to receive challenge status updates
</ParamField>

<ParamField body="rules" type="string[]">
  Game rules/modifiers:
  - `noAbort` - Players cannot abort the game
  - `noRematch` - Players cannot offer a rematch
  - `noGiveTime` - Players cannot give extra time
  - `noClaimWin` - Players cannot claim victory when opponent is disconnected
  - `noEarlyDraw` - Players cannot offer draw before ply 40
</ParamField>

<ParamField body="acceptByToken" type="string">
  OAuth token of the opponent for instant acceptance
</ParamField>

#### Request

<CodeGroup>
```bash cURL
# Create a 5+3 blitz challenge
curl -X POST https://lichess.org/api/challenge/opponent_username \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'rated=true&clock.limit=300&clock.increment=3&color=white'

# Create correspondence challenge
curl -X POST https://lichess.org/api/challenge/opponent_username \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'rated=true&days=3&color=random'

# Chess960 challenge
curl -X POST https://lichess.org/api/challenge/opponent_username \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'rated=false&clock.limit=600&clock.increment=0&variant=chess960'
```

```python Python
import requests

def create_challenge(username, token, rated=False, time_limit=300, 
                     increment=0, color="random", variant="standard"):
    headers = {"Authorization": f"Bearer {token}"}
    url = f"https://lichess.org/api/challenge/{username}"
    
    data = {
        "rated": rated,
        "clock.limit": time_limit,
        "clock.increment": increment,
        "color": color,
        "variant": variant
    }
    
    response = requests.post(url, headers=headers, data=data)
    
    if response.status_code == 200:
        challenge = response.json()
        print(f"Challenge created: {challenge['challenge']['id']}")
        print(f"URL: {challenge['challenge']['url']}")
        return challenge
    else:
        error = response.json()
        print(f"Error: {error.get('error', 'Unknown error')}")
        return None

# Create 3+2 blitz challenge
create_challenge(
    "opponent_username",
    "lip_yourtoken",
    rated=True,
    time_limit=180,
    increment=2,
    color="white"
)

# Create chess960 challenge
create_challenge(
    "opponent_username",
    "lip_yourtoken",
    rated=False,
    time_limit=600,
    increment=5,
    variant="chess960"
)
```

## Open Challenge

### POST /api/challenge/open

Create an open challenge that anyone can accept. Generates unique URLs for white and black.

<Info>
Open challenges are useful for creating challenge links that can be shared publicly or embedded on websites.
</Info>

#### Authentication

Requires OAuth token with `challenge:write` scope, or anonymous access for verified users.

#### Parameters

Same parameters as [Create a Challenge](#create-a-challenge), except no `username` is needed.

<ParamField body="rated" type="boolean" default="false">
  Whether the game should be rated
</ParamField>

<ParamField body="clock.limit" type="integer">
  Clock initial time in seconds
</ParamField>

<ParamField body="clock.increment" type="integer">
  Clock increment in seconds
</ParamField>

<ParamField body="days" type="integer">
  Days per turn (for correspondence, 1-15)
</ParamField>

<ParamField body="variant" type="string" default="standard">
  Chess variant
</ParamField>

<ParamField body="fen" type="string">
  Custom initial position in FEN
</ParamField>

<ParamField body="name" type="string">
  Optional challenge name/description
</ParamField>

<ParamField body="rules" type="string[]">
  Game rules (same as regular challenges)
</ParamField>

#### Request

<CodeGroup>
```bash cURL
curl -X POST https://lichess.org/api/challenge/open \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'rated=false&clock.limit=600&clock.increment=5&name=Practice Game'
```

```python Python
import requests

def create_open_challenge(token, time_limit=300, increment=0, 
                          rated=False, variant="standard", name=None):

        "rated": rated,

        "clock.increment": increment,
        "variant": variant

        print(f"Challenge ID: {challenge['challenge']['id']}")
        print(f"Play as White: {challenge['urlWhite']}")
        print(f"Play as Black: {challenge['urlBlack']}")
        return challenge
    else:
        print(f"Error: {response.json()}")
        return None

# Create open challenge
create_open_challenge(
    "lip_yourtoken",
    time_limit=600,
    increment=5,
    rated=False,
    name="Anyone can join!"
)
```

## Challenge Handling Best Practices

### Auto-Accept Logic

<CodeGroup>
```python Python
import requests
import json

class ChallengeFilter:
    def __init__(self, token):
        self.token = token
        self.headers = {"Authorization": f"Bearer {token}"}
        self.base_url = "https://lichess.org"
    
    def should_accept(self, challenge):
        """Determine if we should accept this challenge"""
        
        # Only standard chess
        if challenge['variant']['key'] != 'standard':
            return False, 'variant'
        
        # Only rated games
        if not challenge['rated']:
            return False, 'casual'
        
        # Check time control
        tc = challenge.get('timeControl', {})
        if tc.get('type') == 'clock':
            limit = tc.get('limit', 0)
            increment = tc.get('increment', 0)
            
            # Only blitz (3-5 minutes)
            if not (180 <= limit <= 300):
                return False, 'timeControl'
        
        # Check opponent rating
        challenger = challenge.get('challenger', {})
        rating = challenger.get('rating', 0)
        
        # Only play within rating range
        if abs(rating - 1600) > 200:  # Our rating ±200
            return False, 'generic'
        
        return True, None
    
    def handle_challenge(self, challenge):
        """Accept or decline a challenge based on criteria"""
        challenge_id = challenge['id']
        should_accept, reason = self.should_accept(challenge)
        
        if should_accept:
            url = f"{self.base_url}/api/challenge/{challenge_id}/accept"
            response = requests.post(url, headers=self.headers)
            
            if response.status_code == 200:
                print(f"Accepted challenge from {challenge['challenger']['name']}")
            else:
                print(f"Failed to accept: {response.json()}")
        else:
            url = f"{self.base_url}/api/challenge/{challenge_id}/decline"
            data = {"reason": reason}
            response = requests.post(
                url, 
                headers={**self.headers, "Content-Type": "application/json"},
                json=data
            )
            
            print(f"Declined challenge (reason: {reason})")

# Usage in event stream
filter = ChallengeFilter("lip_yourtoken")

# When receiving a challenge event:
# filter.handle_challenge(event['challenge'])
```
</CodeGroup>

### Rate Limiting

Be aware of rate limits:
- **Bot concurrent games**: Typically 5 active games maximum
- **Challenge creation**: Limited to prevent spam
- **Challenge to specific user**: Higher rate limit for friends, lower for strangers

### Error Handling

Common errors when creating challenges:
- `"No such user"` - Username doesn't exist
- `"User does not follow you"` - Required for non-friends in some cases
- `"Cannot challenge yourself"` - Self-explanatory
- `"This user does not accept challenges"` - User has disabled challenges
- `"Too many requests"` - Rate limited
