## Get real-time user data

This endpoint reads public data of a user. If the request is authenticated with OAuth2, the user's private data (like email) is also returned.

<Note>
  If the user is not online, the response may be cached for up to 1 minute.
</Note>

### Endpoint

```
GET https://lichess.org/api/user/{username}
```

### Path Parameters

<ParamField path="username" type="string" required>
  The username of the user to fetch
</ParamField>

### Query Parameters

<ParamField query="trophies" type="boolean" default="false">
  Include user trophies in the response
</ParamField>

<ParamField query="profile" type="boolean" default="true">
  Include the user's profile information (like bio, country, links)
</ParamField>

<ParamField query="rank" type="boolean" default="false">
  Include the user's rating rank in each performance type
</ParamField>

<ParamField query="challenge" type="boolean" default="false">
  Include information about whether the user can be challenged (requires OAuth)
</ParamField>

<ParamField query="fideId" type="boolean" default="false">
  Include the user's FIDE ID if available
</ParamField>

### Example Request

```bash
curl "https://lichess.org/api/user/DrNykterstein?trophies=true"
```

### Response

<ResponseField name="id" type="string">
  The user's ID (lowercase username)
</ResponseField>

<ResponseField name="username" type="string">
  The user's display name
</ResponseField>

<ResponseField name="perfs" type="object">
  Performance statistics for each game variant/time control
  
  <Expandable title="Performance object">
    <ResponseField name="games" type="integer">
      Number of rated games played
    </ResponseField>
    
    <ResponseField name="rating" type="integer">
      Current rating
    </ResponseField>
    
    <ResponseField name="rd" type="integer">
      Rating deviation
    </ResponseField>
    
    <ResponseField name="prog" type="integer">
      Progress in the last 12 games

    <ResponseField name="prov" type="boolean">
      Whether the rating is provisional (fewer than 10 games)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="integer">
  Account creation timestamp in milliseconds
</ResponseField>

<ResponseField name="disabled" type="boolean">

</ResponseField>

<ResponseField name="tosViolation" type="boolean">
  Whether the user violated the Terms of Service (only present if true)
</ResponseField>

<ResponseField name="profile" type="object">
  User profile information (if `profile=true` and user has filled it)
  
  <Expandable title="Profile fields">
    <ResponseField name="country" type="string">

    <ResponseField name="location" type="string">
      User's location
    </ResponseField>
    
    <ResponseField name="bio" type="string">
      User's biography
    </ResponseField>
    
    <ResponseField name="firstName" type="string">
      User's first name
    </ResponseField>
    
    <ResponseField name="lastName" type="string">
      User's last name
    </ResponseField>
    
    <ResponseField name="fideRating" type="integer">
      FIDE rating

  </Expandable>
</ResponseField>

<ResponseField name="seenAt" type="integer">
  Last seen timestamp in milliseconds
</ResponseField>

  Time spent playing

    <ResponseField name="total" type="integer">
      Total play time in seconds

      Lost games
    </ResponseField>
    
    <ResponseField name="win" type="integer">

    </ResponseField>
    
    <ResponseField name="bookmark" type="integer">
      Bookmarked games
    </ResponseField>
    
    <ResponseField name="playing" type="integer">
      Currently playing games
    </ResponseField>
    
    <ResponseField name="import" type="integer">
      Imported games
    </ResponseField>

<ResponseField name="streaming" type="boolean">

### Rate Limiting

This endpoint is rate-limited based on:
- Whether the user is online (offline users have cached responses)
- Whether the request is authenticated
- The user's verification status

Authenticated and verified users have higher rate limits.
